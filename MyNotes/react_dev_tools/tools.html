Q
What tools we need to build web application project?
A
Compiler
What is a compiler?
=>Compiler turn new javascript to old javascript
=>A lot of browsers does not support new version of javascript like ES 6 and ES 7 and Typescript

Ex:
Babel
Module Bundlers
What is Module Bundler?
=>Is a tool to bundle all the files into a single file based on their dependencies.

Ex:
Browserify
Webpack
Task Runners
What is task runner?
=>Task runners are tools for defining and running tasks.Task is anything you might to do manually via command line, including compile less to css, publish new build online.etc.

Ex:
Grunt
Gulp
dependency manager
What is dependency manager?
=>Download javascript modules based on dependencies.

Detailed explanation of each tools:

Babel
=>Babel can transfer ES6 to ES5
=>Babel can transfer React JSX
=>Babel can not bundle any files
Browserify
=>Browserify is designed for bundling Node packages for browser.
=>It doesn’t designed to support for CSS, images and fonts. 
=>We can use plugins to make browserify to support css. images and fonts, but it is not encouraged. 
Web pack
=>web pack is a tool that takes javascript modules with dependencies and transfer them into files
=>How does web pack handle css and images since it only takes in javascript?
=>Use loaders
=>loader can take in anything and output anything, meaning it can takes in CSS and output javascript. 
Grunt
=>Grunt is a tool to run task that we defined. 
=>Grunt task is defined by configuration object which are defined decoratively. 
Gulp
=>Like grunt, it is a tool to define and run tasks. 
=>gulp defines tasks by plain old functions.
