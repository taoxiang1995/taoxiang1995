Q
How to import code in one file and use it in another file?
A
=>In ES6 (Ecmascript 2015)
//in foo.js file
export function add (){
     return 1;
}

export function subtract(){
     return 0;
}
//in bar.js file

import {add, subtract} from './foo'
add(); //return 1
subtract(); //return 0

//in bar.js file
import *  as foo from './foo'
foo.add(); //return 1
foo.subtract(); //return 0

Q
When we need to set multiple state in react component, is there any syntax sugar for that?
A
=>Yes
=>we can combine all the states we need to change into one object and pass it to setState function:
Ex:
var stateToBeSet = {
     state1 : val1,
     state2 : val2,
     state3 : val3 
}

this.setState(stateToBeSet);

=>In this case, state1, state2, and state3 will be updated without affecting other states.

Q
Because of the security concerns, IOS will not support http request by default, how to allow http request?
A
Adding this code snippet under: <key>NSExceptionDomains</key> in info.plist file
<key>openweathermap.org</key>
      <dict>
        <key>NSIncludesSubdomains</key>
        <true/>
        <key>NSExceptionAllowsInsecureHTTPLoads</key>
        <true/>
      </dict>
=> When we change the info.plist file, we need to recompile the whole project in Xcode, not just only reload the simulator.

Export & Import

Q
How to export a class, variable or a function?
A
Export some variables, functions and classes
 
export { name1, name2, …, nameN };
export { variable1 as name1, variable2 as name2, …, nameN };
export let name1, name2, …, nameN; // also var
export let name1 = …, name2 = …, …, nameN; // also var, const

Export default
export default expression;
export default function (…) { … } // also class, function*
export default function name1(…) { … } // also class, function*
export { name1 as default, … };
export from another file
export * from …;
export { name1, name2, …, nameN } from …;
export { import1 as name1, import2 as name2, …, nameN } from …;
import 
 
import defaultMember from "module-name";
import * as name from "module-name";
import { member } from "module-name";
import { member as alias } from "module-name";
import { member1 , member2 } from "module-name";
import { member1 , member2 as alias2 , [...] } from "module-name";
import defaultMember, { member [ , [...] ] } from "module-name";
import defaultMember, * as name from "module-name";
import "module-name";


We need web pack to bundle everything together. So ignored to use import and export, we need to have module bundler. 
If we put the file out side the scope of module bundler, it wont work.
