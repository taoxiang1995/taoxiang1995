Q
What is promise?
A
=>promise represents the eventual results of an asynchronies operation.
=>it has 3 states:
     =>pending
     =>fulfilled
     =>rejected

Q
Why do we need to use promise?
A
=>iliminate callback hell.
=>still can guarantee the execution order. (execute something after fetch the result) better way of writing async callback 
=>have more features: all() spread()
=>maintain high performance, fit in with node.js pattern and design thinking (event loop, asynchronies IO)

Q
What is method chaining?
A
Method changing is a syntax sugar to simplify scenario that calling multiple functions on the same object. 
Ex:

not using chaning
var bob = new Kitten();
bob.setName('Bob');
bob.setColor('black');
bob.setGender('male');
bob.save();
using chaning
new Kitten()
.setName('Bob')
.setColor('black')
.setGender('male')
.save();

Q
Can we just chain normal functions?

=>No, we can not. 
we need each previous function to return its calling object in order to use chaining.
Otherwise, .fun() have no calling object.

Q
How to combine promise and chain function to reduce callback?
A
=>Promise API propose following:
  1. Each asynchronies call will return a promise object.
  2. Each promise function can have a then function that can take two arguments: a success handler and an error handler
  3. Then function will also return a promise to allow chaining 
  4. each then function can return a value which will be passed to its chaining function as an argument.
