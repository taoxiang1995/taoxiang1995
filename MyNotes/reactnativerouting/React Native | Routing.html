React Native Router

Q
What is react native router and why should we use it?
A
Why?
It is super easy to use
It can be used in both android and IOS

What?
https://github.com/aksonov/react-native-router-flux
react-native-router-flux is a routing package that allows you to:
  • Define scene transitions in one central location
  • Without having to pass navigator objects around, and allow you to
  • Call transitions anywhere in your code with a simple syntax (e.g. Actions.login({username, password}) orActions.profile({profile}) or even Actions.profile(123) - all params will be part of this.props for given Scene component).

Usage:
https://github.com/aksonov/react-native-router-flux/blob/master/docs/API_CONFIGURATION.md
How does it works?
We register all the scene in the navigationRouter file:
later we can use the functions provided by Action to navigate to or pop to any scene (page)
What is Scene?
=>Scene is like a page or screen.
=>It normally defined in the container folder, using ignite generate screen Settings
=>We need to register scene inside of router in the avigatinruter file:
for each scene, we need to at least procide
(1) A unique string name to refer to
(2) the component name to render (container)
(3) title (a string) appear in the nav bar.
The structure of the definition of the router?
<Router>
  <Scene>
    <Scene></Scene>
    <Scene></Scene>
  </Scene>
</Router>
How to go to a certain page?
Actions.ROUTE_NAME({type: 'reset'});
//pass in the props that this scene needed to render
How to go back to the previoius scene?
Actions.pop()
How does it work with ignite?
