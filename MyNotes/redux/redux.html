Q
What is the difference between presentational component and container component?
A
Presentational Component
=>Concerned with how things look
=>No dependencies on the rest of the app (no actions or store dependency)
=>Receive data exclusively from this.props.
=>Rarely have its own state, (can have UI state)
=>written as functional component
=>have DOM markup and its own state.
Container Component
=>Concerned with how things work
=>Can have some wrapping divs but no style
=>Provide data to presentational component or other container component via props. 
=>Call actions.
=>Generated using higher order component.
=>are often statefull.


Presentational Components		 Container Components
Purpose How things look (markup, styles) How things work (data fetching, state updates)
Aware of Redux No Yes
To read data Read data from props Subscribe to Redux state
To change data Invoke callbacks from props Dispatch Redux actions
Are written By hand Usually generated by React Redux


Q
Why separate presentational component and container component?
A
=>Better reusabiliy
=>Better understanding of the app
=>more flexible, tweak the UI without touching the logic.


Q
How to create a container component?
A
=>The function of a container component is to
(1) read parts of redux state tree and pass it to props of child component.
(2) Receive dispatch method and inject it to child components as props. 
=>How to create a container component?

(1) Make Store available to all the container component:
Ex:
import React from 'react'
import { render } from 'react-dom'
import { Provider } from 'react-redux'
import { createStore } from 'redux'
import todoApp from './reducers'
import App from './components/App'

let store = createStore(todoApp)

render(
  <Provider store={store}>
    <App />
  </Provider>,
  document.getElementById('root')
)

(2) Define mapStateToProps function: 
(Why) do we need mapStateToProps?
map each variable props of wrapping presentational component to part of global state. 
(What) should mapStateToProps function return?
mapStateToProps should return an object containing all the fields of props that you wanna pass to component. 
So the fields names should be exactly the same as props referenced inside of component. 

Ex:
const getVisibleTodos = (todos, filter) => {
  switch (filter) {
    case 'SHOW_ALL':
      return todos
    case 'SHOW_COMPLETED':
      return todos.filter(t => t.completed)
    case 'SHOW_ACTIVE':
      return todos.filter(t => !t.completed)
  }
}

const mapStateToProps = (state) => {
  return {
    todos: getVisibleTodos(state.todos, state.visibilityFilter)
  }
}

(3) Define mapDispatchToProps function: 
(Why) do we need apDispatchToProps function?
sometimes, we need to give presentational component a chance to change the global state. 
Since presentational component doesn’t have direct access to global state, we call actions in props callback functions.
(What) should mapDispatchToProps return
return an object containing the implementation of all the props callback function of wrapping presentational component. 
receives the dispatch() method and returns callback props that you want to inject into the presentational component
Ex:
const mapDispatchToProps = (dispatch) => {
  return {
    onTodoClick: (id) => {
      dispatch(toggleTodo(id))
    }
  }
}

(How) can we define props that has nothing to do with state and actions?
there are two ways:
(1) simply define it in the mapStateToProps:
Ex:
const fooBar = () => 'foobar!'

const mapStateToProps = (state) => {
  return {
    todos: getVisibleTodos(state.todos, state.visibilityFilter),
    fooBar : fooBar
  }
}

(2) pass it to container component, we can use ownProps to reference props passed to container inside of mapStateToProps function.
The good thing is if the props changed, the mapStateToProps will be called again.
Ex:

Some Other Component

 <MyContainer fooBar={() => alert('foobar!')}

MyContainer


const mapStateToProps = (state, ownProps) => {
  return {
    todos: getVisibleTodos(state.todos, state.visibilityFilter),
    fooBar: ownProps.fooBar
  }
}

(4) Define container component by calling connect( ) method from redux:
Ex:
import { connect } from 'react-redux'

const VisibleTodoList = connect(
  mapStateToProps,
  mapDispatchToProps
)(TodoList)

export default VisibleTodoList



Q
How to combine container component and presentational component together?
A
=>(Why) do we need to connect container component with presentational component?
Presentational components only takes in props, so it doesn’t have connection with the global state.
Container component have connection to the state. So we need to connect the props of presentational component (either function or variable) to global state.
=>(How) can we connect container with presentational:
import { connect } from 'react-redux'

const VisibleTodoList = connect(
  mapStateToProps,
  mapDispatchToProps
)(TodoList)

export default VisibleTodoList
=>(How) can we connect multiple presentational components to one container component?
The idea is
(1) wrap up these multiple presentational view with another container component:
import OtherFeature from '../containers/OtherFeature'

export default function ThingView({ entity, doStuff }) {
  if (!entity) return <div>Not Found</div>

  return (
    <div>
      {entity}
      <button onClick={() => doStuff()}>stuff</button>

      <OtherFeature /> // connected to store, independent container (no router props)
    </div>
  )
}

(2) connect this wrapping container with the container:
export default connect(mapState, mapDispatch)(ThingView)

How to figure out state?
Think if this component need any data outside the component to display its view?
(How) to figure out the minimum state, (how to tell if certain piece of data should be state or not)?
Ask three questions:
=>Is it passed in from a parent via this.props?
=>Does it remain unchanged over time?
=>Can you compute it based on any other state or props?
If the answer to any of above question is YES, then it is not 
How to figure out action?
Think what is the behavior of this component?

(Step 5) Write Reducers for each action
const todo = (state, action) => {
  switch (action.type) {
     case ‘ADD_TODO’:
      return […state,{id: action.id, text: action.text, 
              completed:false}]
 
     case ‘TOGGLE_TODO’:
        return state.map(todo =>
                if (todo.id !== action.id) {
                  return todo
                }
                 return Object.assign({}, 
                    todo, {completed: !todo.completed})
            )
 
      case ‘SET_VISIBILITY_FILTER’: {
       return action.filter
      }
 
     default:
      return state
    } 
}

const todos = (state = [], action) => 
{
     switch (action.type)
      {
        case 'ADD_TODO':
           return [ ...state, todo(undefined, action) ]
 
        case 'TOGGLE_TODO':
           return state.map(t => todo(t, action) )
 
        default: return state
     }
} 
export default todos
