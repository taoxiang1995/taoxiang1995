Service

Q
What is service in Angular?
A
Service is a group of javascript functions that are responsible to do a specific tasks only.
If a component want to use service, it needs to inject it.

Why do we need  service?
The idea is component should only be responsible for the view, so we should seperate the businees from it. 
So we put the specific task like get data, calculate something in service. 

Q
How to make a service?
A
https://angular.io/docs/ts/latest/tutorial/toh-pt4.html
(1) import necessary modules and most importantly, especially injectable from '@angular/core'

(2) define service class and export it. Note we need to say that class is injectable.
Naming convention: serviceName.service.ts

Ex:

import { Injectable } from '@angular/core';
import { HEROES } from './mock-heroes';
import { Hero } from './hero';

@Injectable()
export class HeroService {
  getHeroes () {
  return Promise.resolve(HEROES);
  }
}

(2)Configure routes (app.routing.ts)
=>path:  the router matches this route's path to the URL in the browser address bar

import { Routes, RouterModule } from '@angular/router';

import { HeroesComponent }      from './heroes.component';

const appRoutes: Routes = [

    path: 'heroes',
    component: HeroesComponent
  }

export const routing = RouterModule.forRoot(appRoutes);

(3)Make the routes available to all the component by setting it up in the app.module.ts
import { routing }        from './app.routing';

@NgModule({
  imports: [
    BrowserModule,
    FormsModule,

  ],
})

}
(4)Tell where to render component
Putting router-outlet in the template, the hit component will be rendered right below <router-outlet>

 `

How to use http request in angular2?
A
(1)Import necessarty library:

import { Headers, Http, Response } from '@angular/http';
import 'rxjs/add/operator/toPromise';


(2)Register it to constructor:

constructor(private http : Http) {}


(3)Make the http call

getHeroes () : Promise<Hero[]> {
  return this.http.get(this.heroesUrl)
  .toPromise()
  .then(response => response.json().data as Hero[])
  .catch(this.handleError);
  }


  private post (hero : Hero) : Promise<Hero> {
  let headers = new Headers ({
  'Content-Type' : 'application/json'
  });

  return this.http
  .post(this.heroesUrl, JSON.stringify(hero), {headers:headers})
  .toPromise()
  .then(res => res.json().data)
  .catch(this.handleError);
  }

  private put (hero : Hero) : Promise<Hero> {
  let headers = new Headers();
  headers.append('Content-Type', 'application/json');

  let url = `${this.heroesUrl}/${hero.id}`;

  return this.http
  .put(url, JSON.stringify(hero), {headers : headers})
  .toPromise()
  .then(()=>hero)
  .catch(this.handleError);
  }

  delete(hero: Hero): Promise<Response> {
  let headers = new Headers();
  headers.append('Content-Type', 'application/json');

  let url = `${this.heroesUrl}/${hero.id}`;

  return this.http
  .delete(url, {headers: headers})
  .toPromise()
  .catch(this.handleError);
  }

  save(hero: Hero): Promise<Hero> {
  if (hero.id) {
  return this.put(hero);
  }
  return this.post(hero);
  }


Q
How to create a mock server for http request?
A
=>Why (we need to mock)?
Sometimes for developemnt purpose, we do not have a server ready, so we need to mock a server, specifying
what will be returned  when certain end point is hit.

=>How (to mock the server)?
[Basic Idea]
The idea is to replace the backend service and data source with our own service and datasource. 
So whenever a api request is made, it will hit ion our own defined service. 

(1)include necessary modules

import { XHRBackend } from '@angular/http';
import { InMemoryBackendService, SEED_DATA } from 'angular2-in-memory-web-api';
import { InMemoryDataService } from './in-memory-data.service';

(2)register backend and data in provider, and spcecify the new service address

providers: [
  HeroService,
  { provide: XHRBackend, useClass: InMemoryBackendService }, // in-mem server
  { provide: SEED_DATA, useClass: InMemoryDataService } // in-mem server data
  ],

(3)define our own dataservice:

//in in-memory-data.service.ts

export class InMemoryDataService {
  createDb() {
  let heroes = [
  {id: 11, name: 'Mr. Nice'},
  {id: 12, name: 'Narco'},

Q
How does component communicate with each other?
A
=>Why (need to communicate)
Sometimes, when a componend did certain behavior (getting something form the db) it needs to let other related component know, like to update their view.

=>How (to transfer the information)?
[Basic idea] using the @output and EventEmitter
What is output?
opposite to @input, @output  decorator makes the property available as an event binding , like:
<Comp (outputFunction) = "something" />

what is EventEmitter?
EventEmitter will tell if defined event has happened.
  @Output()
  close = new EventEmitter();

goBack(savedHero: Hero = null): void {
  this.close.emit(savedHero);
  if (this.navigated) { window.history.back(); }
  }

Steps:
//in Component:
(1)Include depended module:
import { EventEmitter, Output} from '@angular/core';

(2)Declare and expose the property event:
@Output()
  close = new EventEmitter();

(3)triger the event:
this.close.emit(savedHero);

//in component that uses this component
(1)assign function to event

  <my-hero-detail (close)="close($event)"></my-hero-detail>

(2)Define the function

close(savedHero: Hero): void {
  this.addingHero = false;
  if (savedHero) { this.getHeroes(); }
  }
