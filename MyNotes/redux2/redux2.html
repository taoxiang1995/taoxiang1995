Q
What is Redux?
A
According to its official site:
Redux is a predictable state container for JavaScript (React) apps.
=>Redux provides an easy way to centralize the state of react application.

                                   


Three principles of redux:
(1) Single source of truth: The state of whole application is stored in an object tree within a single store.
console.log(store.getState())

  todos: [

    },
    {
      text: 'Keep all state in a single tree',

    }
  ]

*/
(2) State is read only: The only way to change the state is to emit an action, an object describing what happened. 
store.dispatch({
  type: 'COMPLETE_TODO',
  index: 1
})

store.dispatch({
  type: 'SET_VISIBILITY_FILTER',
  filter: 'SHOW_COMPLETED'
})
(3) Changes are made with pure functions: write reducers to return new state. 

function visibilityFilter(state = 'SHOW_ALL', action) {
  switch (action.type) {
    case 'SET_VISIBILITY_FILTER':
      return action.filter
    default:
      return state
  }
}

function todos(state = [], action) {
  switch (action.type) {
    case 'ADD_TODO':
      return [
        ...state,
        {
          text: action.text,
          completed: false
        }
      ]
    case 'COMPLETE_TODO':
      return state.map((todo, index) => {
        if (index === action.index) {
          return Object.assign({}, todo, {
            completed: true
          })
        }
        return todo
      })
    default:
      return state
  }
}

import { combineReducers, createStore } from 'redux'
let reducer = combineReducers({ visibilityFilter, todos })
let store = createStore(reducer)

Q
Why do we need redux?
A
=>There are so many state and data our code need to manage: server response data, cached data, locally created data and UI state.
=>Model and state will affect each other, some times, it will be really hard to trace the data flow and lost control over the state.
=>increasing requirement for new feature will make app way more complicated to build. 

=>Redux help us to manage the state of or up and make it easy to determine the state of app.

Who can use the redux?

=>develop complex, and heavy state application.
=>can use it for any view library, not only react. 


Redux Basics:

Action
What:
=>Actions are payloads of information that send data from your application to your store. They are the only source of information for the store. You send them to the store using store.dispatch().
=>Action just describes smoothing has happened, but doesn’t specify how application’s state should change. 
=>Actions are plain JavaScript objects. Actions must have a type property that indicates the type of action being performed. Types should typically be defined as string constants.

We use action creator to create actions.
action creators simply returns an action object.

=>Reducer specifies how the application’s state changes in response
=>Reducer is a pure function that takes the previous state and an action, returns the next state.

Q
How ( can we create a new state without mutating the old state )?:
A
=>Use Object.assign( ) function
=>It will merge the second and else elements to the first argument.
=>if there is duplicate keys, take the last key value.
Ex:
Object.assign({}, state, {
        visibilityFilter: action.filter
      })
//In this case, it will combine the old state and new visibilityFilter, and assign it to an empty
//object, and return the combined object.

Q
How to handle multiple actions inside of one reducer?
A
=>Use switch statement. 
Ex:
function todoApp(state = initialState, action) {
  switch (action.type) {
    case SET_VISIBILITY_FILTER:
      return Object.assign({}, state, {
        visibilityFilter: action.filter
      })
    case ADD_TODO:
      return Object.assign({}, state, {
        todos: [
          ...state.todos,
          {
            text: action.text,
            completed: false
          }
        ]
      })   
    default:
      return state
  }
}


Q
Since if we put too many cases inside one reducer will be too verbose, is it possible to divide the cases into multiple reducers?
A
=>Yes, we can, if the state each case manages don’t interdepend on each other.
=>We can create sub reducers that only handle one of some states, and combine them together. 
=>How (to combine the state together)?
two ways:
(1) Use a super reducer function
function todoApp(state = {}, action) {
  return {
    visibilityFilter: visibilityFilter(state.visibilityFilter, action),
    todos: todos(state.todos, action)
  }
}
create reducers to handle each state separately, can in the parent reducer, calling each corresponding function for each state. 

(2) Use combineReducer
since the state name and function name is the same (by convention). so we can use a combineReducer method to put everything together:
import { combineReducers } from 'redux'

const todoApp = combineReducers({
  visibilityFilter,
  todos
})
export default todoApp


Data Flow 

Q
What is the data flow of redux?
A
(1) UI triggered an action by calling store.dispatch(action)
(2) Store calls the reducer with current state and the action (Root reducer may combine multiple reducer to one state tree).
(3) Store update the current state tree with the state tree returned by reducer. (registered listener will be invoked at this time)
