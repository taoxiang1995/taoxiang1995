Q
What is ignite and why should we use it?
A
=>Ignite is a react native framework to help develop react native app.
=>included a lot of common react native libraries:

  • Included Common Libs:
    • react-native-vector-icons
    • react-native-animatable
    • react-native-i18n
    • react-native-drawer
    • apisauce
    • reduxsauce
    • react-native-maps
    • rn-translate-template
  • Included Developer Libs:
    • reactotron
    • AVA
    • enzyme
    • react-native-mock
    • mockery
    • nyc

=> build in generators:

Generate an app
ignite new MyApplication
Generate a component
ignite generate component SmoothButton
Generate a container
ignite generate container SubMenu
Generate a screen
ignite generate screen Settings
Generate a listview
ignite generate listview HotDates
Generate a mapview
ignite generate mapview MapScreen
Import images
ignite import imagesfile


Q
What is the basic file structure of ignite?
A

App folder:
Components:
define UI components
Containers:
define pages and containers (redux)
img:
where to put all the images
Lib:
proving ground for components that could be reuseable outside your project.
Navigation:
Where to define navigation bar (in NavItem.js)
Define routers (in NavigationRouter.js)https://github.com/aksonov/react-native-router-flux
Define NavigationDrawer (in NaviationDrawer.js) https://github.com/root-two/react-native-drawer
Themes
Application specific themes
* Base Styles
* Fonts
* Metrics
* Colors

Test folder:

write tests


Starting Process of the code:

Where is the start point of the app?
index.os.js
OR
index.android.js
Where is the root component?
App/Containers/App.js
`App.js` is first component loaded after `index.ios.js` or `index.android.js`.  The purpose of this file is to setup Redux or any other non-visual "global" modules.  Having Redux setup here helps with the hot-reloading process in React Native during development as it won't try to reload your sagas and reducers should your colors change (for example).
Where to create store and connect redux to provider.
Where is the first visual component?
`RootContainer.js`
is the first visual component in the app.  It is the ancestor of all other screens and components.




Q
What is screen and how to create a screen?
A
A container is what they call a "Smart Component" in Redux.  It is a component
which knows about Redux.  They are usually used as "Screens".
